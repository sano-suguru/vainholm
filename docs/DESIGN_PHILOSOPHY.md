# 設計哲学 — Vainholmの思想的基盤

> **「引き算の美学」×「制約の緊張感」×「ハクスラの快感」**

本ドキュメントは、Vainholmの設計判断の背景となる哲学を記述する。
具体的な仕様については [GAME_DESIGN.md](./GAME_DESIGN.md) を参照。

---

# 第1部：Brogueのゲームデザイン哲学

Brogueは Brian Walker が開発したローグライクゲームで、ゲームデザインの教科書とも言える傑作です。その哲学と設計を具体的に解説します。

---

## 1. 核心哲学：「Depth from Simplicity」（シンプルさから深さを）

Brogueの最大の特徴は、**要素を極限まで削ぎ落としながら、戦略的深さを最大化している**ことです。

### 具体例：ステータスの排除

| 一般的なRPG | Brogue |
|-------------|--------|
| STR, DEX, INT, WIS, CON… | **なし** |
| レベルアップで能力値振り分け | レベルアップ自体がない |
| 複雑なスキルツリー | 装備とアイテムのみ |

**なぜこれが優れているか？**

プレイヤーの意思決定が「数値の最適化」ではなく「状況判断」に集中します。「このフロアで火炎の杖を使うべきか、温存すべきか」という**文脈依存の判断**がゲームの核になります。

---

## 2. 「Emergent Gameplay」（創発的ゲームプレイ）

Brogueは少数のシンプルなルールの**相互作用**から、無限のシチュエーションを生み出します。

### 具体例：火・ガス・水の相互作用

```
【基本ルール】
・火は草を燃やす
・燃える草は煙を出す
・ガスは火で爆発する
・水は火を消す
・沼は移動を遅くする

【創発する状況】
→ 草原で敵に囲まれた！
→ 火炎の杖で草を燃やす
→ 煙で視界が遮られ、敵の遠距離攻撃を防ぐ
→ 煙に紛れて逃走
→ だが追ってきた敵の持つ松明が、逃げ込んだ部屋のガスに引火して…
```

このような**計画的にも偶発的にも起こる連鎖反応**が、毎回のプレイをユニークにします。

---

## 3. 「Transparent Mechanics」（透明なメカニクス）

Brogueは**すべての情報をプレイヤーに開示**します。

### 具体例：ダメージ計算

```
敵をマウスオーバーすると表示される：
「この攻撃で倒せる確率: 74%」
「被ダメージ予測: 12-18」
```

**設計意図：**

- 「運ゲー」ではなく「情報に基づく意思決定」
- 死んだとき「なぜ死んだか」が明確
- **Informed Decision（情報に基づく決断）** を可能にする

これは「難しいけど理不尽ではない」という体験を生みます。

---

## 4. 「Meaningful Scarcity」（意味のある希少性）

リソースが常にギリギリで、すべての選択に重みがあります。

### 具体例：強化の巻物システム

```
【Enchant Scroll（強化の巻物）】
・ゲーム全体で約12〜15枚しか手に入らない
・武器に使う？防具に使う？杖に使う？
・一度使うと取り返しがつかない

【結果生まれるジレンマ】
「この+2の剣を+3にするか、それとも後でもっと良い武器が出るかも…」
```

**これにより：**

- 毎回の強化が「イベント」になる
- ビルドの方向性を決める重大な選択になる
- 「最適解を探す」のではなく「今あるもので最善を尽くす」ゲームになる

---

## 5. 「Ally System」— シンプルなのに深い仲間システム

Brogueの仲間システムは、驚くほどシンプルながら戦略的です。

```
【基本ルール】
・モンスターを魔法で「味方化」できる
・味方も強化の巻物で育てられる
・味方が死んだら、投資したリソースは失われる

【生まれる戦略】
1. 「自分を強化するか、仲間を強化するか」
2. 「仲間を前線に出すか、温存するか」
3. 「この仲間は見捨てて逃げるか、助けるために危険を冒すか」
```

数値管理ではなく**感情的な絆と戦略的判断の葛藤**が生まれます。

---

## 6. 「Visual Clarity」（視覚的明瞭さ）

ASCIIベースながら、**色と光の表現**が秀逸です。

```
【光源システム】
・松明の光が壁に反射する
・暗闘では敵が見えない
・光源を消せばステルスできる
・だが自分も見えなくなるリスク

【色の使い方】
・赤 = 危険（溶岩、火）
・青 = 水、冷気
・緑 = 毒、酸
・黄 = 光源、宝
```

シンプルな記号でも**一目で状況を把握できる**設計です。

---

## 7. Brogueのゲームデザイン原則まとめ

| 原則 | Brogueの実装 | 効果 |
|------|-------------|------|
| **Less is More** | ステータス排除 | 判断に集中できる |
| **Systemic Design** | 要素の相互作用 | 創発的な体験 |
| **Transparency** | 確率の明示 | 公正な難易度 |
| **Meaningful Choice** | リソースの希少性 | 選択の重み |
| **Risk/Reward** | 深層ほど良い報酬 | テンションの管理 |
| **No Grinding** | レベルなし | スキルで勝負 |

---

> **「複雑さは深さではない。深さはシンプルな要素の豊かな相互作用から生まれる」**

Brogueは「何を入れるか」ではなく「何を入れないか」を極めたゲームです。すべての要素が意味を持ち、すべての選択が結果に繋がり、すべての死から学べる。

これこそが、ローグライクの、そしてゲームデザインの理想形の一つと言えるでしょう。

# 第2部：片道勇者のゲームデザイン哲学

『片道勇者』は SmokingWOLF が2012年に開発したローグライクRPGです。「強制横スクロールRPG」という未開拓の領域を開拓し、**制約を武器に変える**設計思想でジャンルに新風を吹き込みました。

---

## 8. 開発の出発点：「未開の領域への確信」

SmokingWOLFは2011年11月頃、ゲームのスクロール要素についての議論の中で「強制横スクロールRPG」という概念に着想を得ました。調査の結果ほぼ未開の分野であることがわかり、「必ずおもしろく作れるはずだ」という確信のもと開発を開始しています。

**この姿勢自体が設計哲学を示しています：**

- 既存の組み合わせではなく、構造的な新規性を追求
- 「できるかどうか」ではなく「面白くできる確信」を重視
- 一つの核心的アイデアを軸に全体を構築

---

## 9. 核心哲学：「Constraint as Design」（制約を設計に）

片道勇者の最大の特徴は、**「できないこと」を増やすことで「できること」の価値を上げている**点です。

### 「闇」というメカニクス

```
【基本ルール】
・画面左側から「闇」が世界を喰らい続ける
・闇に飲まれればゲームオーバー
・引き返すことはできない
・街で用事をすませている間も、戦っている間も、世界は強制的にスクロールされる
```

### Brogueとの対比

| 要素 | Brogue | 片道勇者 |
|------|--------|----------|
| 時間 | 無制限（ターン制） | **制限あり**（闇の進行） |
| 探索 | 自由に戻れる | **一方通行** |
| 最適解 | 「全部調べる」が正解になりがち | 「取捨選択」が必須 |
| 判断の先送り | 可能 | **不可能** |

**なぜこれが優れているか？**

従来のローグライクでは「時間をかければ最適解に近づける」という構造がありました。片道勇者はこれを破壊し、**すべての行動に時間コストを課す**ことで、常に不完全な情報での決断を強います。

---

## 10. 「Opportunity Cost Visualization」（機会費用の可視化）

片道勇者は経済学の概念である「機会費用」をゲームメカニクスとして体現しています。

### 具体例：典型的な死因

```
【公式が挙げる典型的死因】
・ダンジョンに突入したものの脱出が間に合わなくなり、画面端に飲み込まれた
・ぎりぎり間に合うと思ってアイテムを拾いに行ったら、画面端に飲み込まれた
・前から来た敵と長期戦を繰り広げていたら、敵と画面端に挟まれて逃げ場がなくなった
・買い物していたら街の入口がスクロールにのまれて出られなくなった
```

### 設計意図の分析

これらの状況は**偶然ではなく必然的に発生する**よう設計されています：

| 状況 | トレードオフ |
|------|-------------|
| ダンジョン探索 | 報酬 vs 脱出リスク |
| アイテム回収 | 即時利益 vs 時間コスト |
| 戦闘継続 | 経験値 vs 位置の悪化 |
| 買い物 | 装備強化 vs 時間消費 |

**すべての行動が「何かを得て何かを失う」構造**になっています。

---

## 11. 「Visible Loss」（見える喪失）の心理設計

片道勇者は「喪失」の見せ方に工夫があります。

### 二種類の喪失

| 喪失タイプ | プレイヤーの認識 | 心理的効果 |
|-----------|----------------|-----------|
| 視界外で失う | 気づかない | ストレスなし |
| **視界内で失う** | はっきり認識する | 強烈な後悔 |

### 設計効果

「見えているのに取れなかった」という体験は：

- **自分の判断ミス**として認識される（理不尽ではない）
- 「次こそは」というリプレイ動機を自然に生成
- 上達の実感につながる（同じ失敗をしなくなる）

これはBrogueの「Transparency（透明性）」と通じる哲学です。死因が明確だからこそ、学びがある。

---

## 12. 「Meta Progression」（メタ進行）— ローグライトの先駆

片道勇者はパーマデス（永久死）でありながら、**周回による成長要素**を持ちます。

### 持ち越しシステム

```
【基本ルール】
・死亡したらレベル１からやり直し
・ただし死亡時にいくつかのアイテムを倉庫に預けられる
・クラス解放や装備の持ち越しといった周回プレイの要素がある
```

### 多層的な目標構造

| 目標レイヤー | 内容 | モチベーション |
|-------------|------|---------------|
| 短期 | 今回のランで生き延びる | 即時の緊張感 |
| 中期 | 新クラス・アイテム解放 | 収集欲・探究心 |
| 長期 | 真エンディング到達 | 物語的達成感 |

**「死んでも完全に無駄ではない」**という感覚が、ハードコアゲーマーとカジュアル層の両方を取り込む設計になっています。

この構造は後の「ローグライト」ジャンル（Hades、Dead Cellsなど）の定番となりましたが、2012年時点では先進的でした。

---

## 13. 「Seed Sharing」（シード共有）— 孤独に社会性を

### システム

```
マップはランダムではなく、文字列によって生成される
→ 以前と同じマップや、他の人がプレイしたマップで遊ぶことも可能
→ 公式でランキングも存在
```

### 設計効果

| 効果 | 説明 |
|------|------|
| 情報共有 | 「このシードは序盤に良い武器がある」等の口コミ |
| 公平な競争 | 同条件でのスコアアタックが可能 |
| コミュニティ形成 | 一人用ゲームに「ゆるい競争」が生まれる |
| リプレイ価値 | 同じ世界を違う戦略で攻略する楽しみ |

**孤独なローグライク体験に社会性を注入する装置**として機能しています。

---

## 14. 「Session Design」（セッション設計）— 30分の魔法

### 事実

```
1ゲーム平均30分ほどで遊べる
```

### なぜ30分が優れているか

| 時間設計 | 効果 |
|---------|------|
| 短すぎない | 達成感がある |
| 長すぎない | 失敗しても「もう一回」が心理的に軽い |
| 隙間時間対応 | アクセスしやすい |
| 強制終了装置 | 闇がゲームを自然に終わらせる |

**強制スクロールは「ゲームを終わらせる装置」**でもあります。だらだら続けられない構造が、プレイ時間を自然に制御し、「あと一回」のループを促進します。

---

## 15. 「Ludonarrative Harmony」（ルドナラティブの調和）

片道勇者は、ゲームルールと物語が完全に一致しています。

### 窓の杜大賞2012の評価

> 「世界が闇に飲まれていくという物語とゲームシステムの融合も見事」

### ルールと物語の対応

| ゲームルール | 物語的意味 |
|-------------|-----------|
| 戻れない | 世界が消えているから |
| 急がなければならない | 闇が迫っているから |
| 魔王を倒す | 闇を止める唯一の方法だから |
| 仲間が闇に飲まれる | 彼らも世界の一部だから |

**ルールが世界観を説明し、世界観がルールを正当化する。**

この「ルドナラティブ・ハーモニー」は、プレイヤーの没入感を高め、ゲームプレイ自体が物語体験になる設計です。

---

## 16. 片道勇者のゲームデザイン原則まとめ

| 原則 | 片道勇者の実装 | 効果 |
|------|---------------|------|
| **Constraint as Design** | 強制スクロール | すべての選択に重み |
| **Opportunity Cost** | 時間 = リソース | トレードオフの連続 |
| **Visible Loss** | 視界内での喪失 | 後悔→リプレイ動機 |
| **Meta Progression** | 持ち越しシステム | 死の無駄化防止 |
| **Seed Sharing** | 文字列マップ生成 | 社会性の注入 |
| **Session Design** | 30分/1プレイ | リプレイしやすさ |
| **Ludonarrative Harmony** | ルールと物語の一致 | 没入感 |

---

> **「できないことを増やすことで、できることの価値を上げる」**

片道勇者は「自由度を上げる」という常識に逆らい、**制約を武器に変えた**ゲームです。一つの核心的アイデア（強制スクロール）から、すべての設計が論理的に導かれている。

これは「引き算の美学」とは異なるアプローチですが、**「一つの制約がすべてを変える」**という意味で、同じく本質的なゲームデザインの理想形と言えるでしょう。

# 第3部：ハイブリッド設計 — 三つの哲学の統合

## 設計コンセプト

**「Brogueの創発的深さ」×「片道勇者の制約による緊張感」×「ハクスラの厳選快感」×「小規模パーティの連携」**

三つの設計哲学を統合し、それぞれの強みを活かしたハイブリッドを目指します。

### 各要素の役割

| 出典 | 取り入れる要素 | 担う役割 |
|------|---------------|---------|
| Brogue | 創発的システム、透明性、希少性 | **深さ** |
| 片道勇者 | 時間制約、メタ進行、セッション設計 | **緊張感とリプレイ性** |
| ハクスラ | プレミアムシステム、厳選 | **収集欲と長期モチベーション** |

## 18. 三つの哲学との整合性チェック

### Brogueの原則との整合

| Brogueの原則 | 本設計での対応 |
|-------------|---------------|
| **Less is More** | プレミアム種類を厳選（武器8種+防具4種+緑5種）、パーティ最大4人 |
| **Emergent Gameplay** | 属性プレミアム×環境（火炎付与で草を燃やす等）の相互作用 |
| **Transparency** | プレミアムは全て可視化、ドロップ時に光の色で判別可能 |
| **Meaningful Scarcity** | 強化の巻物でプレミアム追加可能、でも巻物は希少 |
| **Informed Decision** | 「この装備を誰に？」「仲間を前に出すリスクは？」の判断 |

### 片道勇者の原則との整合

| 片道勇者の原則 | 本設計での対応 |
|---------------|---------------|
| **Constraint as Design** | 崩壊システムによる時間制約 |
| **Opportunity Cost** | 「留まるか進むか」「取りに戻るか諦めるか」の判断 |
| **Visible Loss** | 崩壊に飲まれる装備・仲間が「見える」 |
| **Meta Progression** | 遺産ポイントによるクラス・装備解放 |
| **Session Design** | 崩壊が自然にゲームを終わらせる（30-45分） |
| **Ludonarrative Harmony** | 「崩壊するダンジョンからの脱出」という物語 |

### ハクスラの原則との整合

| ハクスラの原則 | 本設計での対応 |
|---------------|---------------|
| **Loot Hunt** | プレミアムのランダム性と厳選 |
| **Build Diversity** | プレミアムの組み合わせによるビルド |
| **Progression Feel** | 「良い装備が出た！」の快感 |
| **Long-term Goal** | ユニーク収集、メタ進行 |

## 19. 想定されるプレイ体験

### 序盤（1F〜8F）

- プレミアムなし or 低品質装備でスタート
- 「お、かず+1の剣だ！」→ 小さな喜び
- 仲間を1人仲間にする。装備をどう分けるか悩む
- 崩壊はまだ始まっていない。探索に余裕がある

### 中盤（9F〜17F）

- 良プレミアム装備がちらほら
- 「火炎付与の斧と、冷気付与の剣、どっちを主力に？」
- 仲間が2人に。前衛・後衛の役割分担が明確に
- **崩壊開始**。「あのフロアの宝箱、取りに戻るか…？」
- **仲間の死**を初体験。装備ロストの痛みを知る

### 終盤（18F〜25F）

- 神器級を求めて厳選開始
- ユニーク装備を入手。デメリットとどう向き合うか
- 「この仲間に最高の装備を持たせたい。でも死んだら…」
- 崩壊が迫る中、**最後の判断**の連続
- **感情的な愛着**と**戦略的判断**の葛藤

### ラン終了後

- 遺産ポイント獲得
- 「次は騎士クラスを解放しよう」
- 「あのシードでもう一回やってみよう」
- 「30分だし、もう一回…」

## 20. 追加検討事項

### オプション案（入れるか要検討）

| 案 | メリット | デメリット |
|----|---------|-----------|
| **プレミアム継承** | 仲間死亡時、装備の一部が残る | ロストの重みが減る |
| **プレミアム合成** | 2つの装備から1つを作る | 複雑化、バランス調整困難 |
| **呪いプレミアム** | 外せないデメリット付き装備 | Brogueに既存の呪いと被る |
| **仲間専用プレミアム** | 特定の仲間種族にのみ効果 | 判断材料が増えすぎる |
| **崩壊遅延アイテム** | 崩壊を一時的に止める | 緊張感が薄れる可能性 |

# 結論

この設計は三つの哲学を統合することで、以下の体験を目指します：

1. **「あと一回だけ」** の中毒性（プレミアム厳選 + 30分セッション）
2. **「この仲間を守りたい」** の感情（パーティシステム + 永久ロスト）
3. **「この状況をどう切り抜ける？」** の判断（Brogue本来の強み）
4. **「今、決断しなければ」** の緊張感（崩壊システム）
5. **「次こそは」** のリプレイ動機（メタ進行 + 見える喪失）

---

> **「シンプルな要素が、制約の中で、無限の物語を生む」**
> 
> — Brogueの創発性、片道勇者の制約、ハクスラの収集欲。三つの哲学が交わるとき、新しい深さが生まれる。
